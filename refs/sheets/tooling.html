陸Tooling

よく使う道具の起動テンプレートと、再利用しやすいスニペット。加えて、公開ノートへ落とす前のサニタイズとコミット衛生を定型化する。パスは環境変数・プレースホルダで一般化する。 前提: Kali 系。$TOOLS は各自のツール置き場、impacket は venv 優先。

01環境environment

再現性のため、道具は固定パスに集約し、Python 系は venv に隔離する。impacket は system と venv で版が割れやすいので、venv の実体を優先してパスを固定する。

zsh
# 置き場と venv（例）
export TOOLS="$HOME/Tool"
source ~/venvs/impacket/bin/activate

# impacket は venv の実体を直呼び（版ズレ回避）
alias secretsdump='~/venvs/impacket/bin/secretsdump.py'
alias getst='~/venvs/impacket/bin/getST.py'
tipssystem と venv で挙動が違うときは which -a secretsdump.py で実体を確認し、フルパスで固定する。

02起動テンプレートlaunch

頻用ツールの最短起動形。値はすべてプレースホルダ。

bash
# 資格情報の面を測る
nxc smb <TARGETS> -u '<USER>' -p '<PASS>'

# ファイル配布用の一時 HTTP
python3 -m http.server 8000

# ligolo-ng（攻撃側 proxy）
sudo ip tuntap add user $USER mode tun ligolo; sudo ip link set ligolo up
$TOOLS/ligolo/proxy -selfcert

# chisel（reverse socks）
$TOOLS/chisel server -p 8080 --reverse
防御 / countermeasures
  • これらは検証側の起動形。防御視点では、一時 HTTP 配布や reverse socks の外向き接続を egress で検知する。

03スニペットsnippets

LHOST を動的取得(tun0)

bash
export LHOST=$(ip a show tun0 | grep 'inet ' | awk '{print $2}' | cut -d/ -f1)
echo "$LHOST"

待受

bash
# TTY を含むリスナ運用は必要に応じて stty raw -echo で調整
rlwrap nc -lvnp 443
tipsワンライナは環境変数($LHOST 等)で持ち、値の直書きを避ける。公開ノートに実値を残さない習慣づけになる。

04サニタイズ / コミット衛生publish pipeline

private vault → 検証再現 → サニタイズ → 公開、の一方向を守る。公開前に実値(IP・FQDN・ハッシュ・パス・利用者名)を機械的に当て、コミットは著者情報とタイムゾーンを固定する。

bash
# 公開前サニタイズ検査
grep -rInE '([0-9]{1,3}\.){3}[0-9]{1,3}|@gmail\.com|C:\\Users\\[A-Za-z0-9._-]+|/home/[a-z]+/[A-Za-z0-9._-]+' . \
  --include='*.html' --include='*.md' --include='*.js'

# コミット衛生（著者=noreply, TZ=UTC）
git config user.email '<ID>+<HANDLE>@users.noreply.github.com'
export TZ=UTC
git log -3 --format='%an <%ae> | %ai'   # 全行 noreply / +0000 を確認
重要denylist(実識別子の列挙)は公開リポジトリに絶対に含めない。それ自体が deanonymization 材料になる。
要点 / key takeaways
  • 道具は固定パス+venv 隔離で再現性を確保。impacket は版ズレ回避に venv 実体を直呼び。
  • ワンライナは環境変数で持ち、実値の直書きを避ける。これが公開時のサニタイズを楽にする。
  • 公開前は grep で実値を機械検査し、コミットは noreply + TZ=UTC を固定。denylist は公開しない。
